🏗️ How the matrix works

Each scenario applies one (or several) mutations to a baseline of 150 orders routed to 20 technicians. All 8 backends solve the same input. We compare the resulting routes for assignment rate, distance, late minutes, overtime, and runtime.

Scenario Categories

BASE  ·  the baseline solve. No mutations. Used as the starting point for every other scenario.

EAS · 4 scenarios  ·  Execution-Aware Scheduling. Customer-side changes: time windows, service times, depot preferences.

RT · 7 scenarios  ·  Real-Time disruptions. Operations chaos: sick techs, urgent orders, cancellations, shift delays.

GB · 6 scenarios  ·  Goal-Based optimization. Goal weight overrides for travel, SLA, overtime, customer priority.
